# 🎀 Telegram-бот «Настюшкинс» (AI-Companion)

Асинхронный Telegram-бот, имитирующий живое, неформальное общение с девушкой. Работает полностью локально, сохраняет историю диалога в базу данных и не требует платных внешних API.

## 🛠 Технологический стек

* **Язык:** Python 3.10+
* **Фреймворк:** `python-telegram-bot` (v20.7) — асинхронный инструмент для работы с Telegram Bot API.
* **LLM Среда:** `Ollama` (latest) — локальный сервер для запуска больших языковых моделей.
* **Модель:** `Gemma 2 9B` (в формате GGUF) — основа диалоговой логики.
* **База данных:** `SQLite3` (встроенная библиотека Python) — персистентное хранение истории чатов.
* **Конфигурация:** `python-dotenv` — управление секретами через файлы среды.

---

## 🏗 Архитектура и ключевые фичи

Проект спроектирован по принципу разделения слоёв (изоляция БД, логики Telegram и инференса):

1. **Персистентная память (SQLite3):** Бот не сбрасывает контекст при перезапуске процесса. Все сообщения и метаданные пользователей сохраняются в файл `nastya_memory.db`.
2. **Управление контекстом:** Чтобы не переполнять контекстное окно LLM, бот динамически подгружает только последние 10 сообщений для каждого пользователя.
3. **Асинхронный Non-blocking дизайн:** Запросы к Ollama выполняются в пуле потоков через `asyncio.to_thread`, что предотвращает зависание всего бота при генерации ответа одному пользователю.
4. **Отказоустойчивость:** Реализован жесткий таймаут на генерацию (45 секунд). В случае сбоя или зависания модели бот отправляет пользователю поддерживающее сообщение в своем "каноничном" стиле.
5. **Имитация присутствия:** Во время генерации ответа в чате отображается статус `typing` («печатает...»).

### 1. Подготовка модели (Ollama)

Убедитесь, что у вас установлен и запущен Ollama Server.


## 💾 Схема базы данных

Данные хранятся в двух связанных таблицах:

### Таблица `users`

* `user_id` (INTEGER, PK) — Уникальный ID пользователя из Telegram.
* `first_seen` (TIMESTAMP) — Время первого взаимодействия с ботом.
* `last_active` (TIMESTAMP) — Время последнего сообщения.

### Таблица `messages`

* `id` (INTEGER, PK AUTOINCREMENT) — Идентификатор сообщения.
* `user_id` (INTEGER, FK) — Ссылка на пользователя.
* `role` (TEXT) — Роль в диалоге (`user`, `assistant`, `system`).
* `content` (TEXT) — Текст сообщения.
* `timestamp` (TIMESTAMP) — Время отправки.

## 🤖 Логика Персоны (System Prompt)

Личность Насти жестко зашита в системном промпте модели. Основные правила поведения:

* Полный запрет на признание себя искусственным интеллектом.
* Общение исключительно на русском языке (при попытке говорить на иностранном — «включает дурочку» и просит писать понятно).
* Динамическая адаптация стиля общения в зависимости от пола собеседника.
* Удержание длины ответа в пределах 3–5 предложений, завершая реплику открытым вопросом для продолжения беседы.

## 🛠 Доступные команды в чате

* `/start` — Инициализация сессии, первичное знакомство.
* `/reset` — Полная очистка истории сообщений в базе данных для текущего `user_id` (сброс контекста).
